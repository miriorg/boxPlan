## 1. プランの評価基準「サイズの分散度」について

プランの評価基準として「ボックスのサイズが適当に分散していることが望ましい」といただきました。この「分散度」をどのように評価するのが理想に近いか、以下からお選びいただけますでしょうか。

- **A案:** できるだけ多くの**種類**のボックスを使うプランを高評価とする。
  （例：高さが3種類 [H18, H23, H30] を使っているプランは、2種類 [H18, H23] しか使っていないプランより高評価）

- **B案:** 各サイズのボックスが**均等に近い個数**で使われているプランを高評価とする。
  （例：H18が5個, H30が4個のプランは、H18が8個, H30が1個のプランより高評価）

- **C案:** 特に計算上の評価はせず、活用率と個数が同程度のプランが複数あった場合に、見た目のバランスが異なるものを**複数提示**するだけで良い。

**【ご回答】**
>C案
結局は利用者ごとに事情・感じ方が異なるので複数提案できればそれでよい。
編集機能がついているので、ユーザーに直接指定させた方が良い。

## 2. プランの手動調整機能について

「イメージ上のブロックをクリックしたら（中略）その列・その行の高さ・幅を更新する」機能の具体的な動作について、認識を合わせさせてください。

例えば、下図の「B」のボックスをクリックして、より**背の高い**ボックスに変更した場合、どのような更新が行われるのが望ましいですか？

```
  【列1】 【列2】
+-------+-------+
|   A   |   C   |  (行1)
+-------+-------+
|   B   |   D   |  (行2)
+-------+-------+
```

- **A案:** 「B」のボックスだけが背の高いものに変わる。（※この場合、グリッド状のレイアウトが崩れる可能性があります）
  ```
    【列1】 【列2】
  +-------+-------+
  |   A   |   C   |
  +-------+
  |       |
  | B(高) |
  |       |
  +-------+
          |   D   |
          +-------+
  ```

- **B案:** 「B」が含まれる**行全体（行2）**が、新しいボックスの高さに揃えられる。
  （この時、「D」も同じ高さのボックスに自動的に変わる）
  ```
    【列1】 【列2】
  +-------+-------+
  |   A   |   C   |
  +-------+-------+
  |       |       |
  | B(高) | D(高) |
  |       |       |
  +-------+-------+
  ```
  **B案の場合の追加質問:** もし行の高さが高くなった結果、全体の高さが入力されたスペースの高さを超えてしまう場合、どうしますか？
    - `B-1:` 変更できないようにする（エラーメッセージを表示）
    - `B-2:` 超えてしまっても良い（レイアウトは更新する）

- **C案:** その他（ご希望の動作があれば具体的にお聞かせください）

**【ご回答】**
>動きの選択はB-2で、スペースのサイズを超えた場合、レイアウトイメージpaneにエラーである旨をする

## 3. 制作中のテスト環境について

「制作中のテスト環境」についてご要望をいただきました。

テストが終われば、その変更を本番環境に反映させる、という流れになります。
