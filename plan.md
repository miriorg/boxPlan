# 依頼内容

vercel, Githubを利用したWebアプリ(w/Javascript)を作成してください
vercel, Githubともに無料枠内で作成・運用する

## アプリの仕様

ユーザーが入力したスペースの寸法（高さ・幅・奥行き）に収まるブロック型の収納ボックス(ボックスの仕様は後述)の組み合わせプランを提案するアプリ

## 提案するプランの内容

- ボックスの高さ・幅・奥行きとその数
- 積み上げる例
- 積み上げたイメージ（SVGを使いボックスを並べた正面図(高さ・幅を比率で繁栄したもの)

## ボックスの仕様
収納ボックスのサイズの単位はcmです。
アプリではmmでサイズを表示してください。

## プランにおける制約

- プランを作るときはメーカーは統一する
- プランを作るときはボックスの奥行は統一する
- プランでは列毎に幅を統一する
- プランでは行毎に高さを統一する
- プランは奥行き方向は１つのみとする
- プランは高さ方向を、ケースの高さを組み合わせ、入力された高さ以内とする
- プランは幅方向を、ケースの幅を組み合わせ、入力された幅以内とする

## アプリケーションの動作概略
アプリケーションはサイズ入力paneから始まります。
1. サイズ指定paneで必要情報を入力
2. 開始ボタンをクリック
3. 入力された情報からプランを作成
4. レイアウトイメージpaneに作成プランに沿って積み上げた正面図を表示
5. レイアウトリストpaneに、プランで選定した収納ボックスのサイズと個数を表形式一覧する
   - 表にはメーカー・高さ・幅・奥行き・個数を表示する


## サイズ指定paneについて
- 画面には高さ・幅・奥行きを入力する入力欄・「開始」ボタンのフォームがある
- 「1000」,「500」,「100」,「50」,「10」（いずれか一つ有効）の有効無効を表すボタンセットと「＋」、「ー」、「リセット」のボタンを用意し、寸法入力欄（高さ、幅、奥行）の直下にそれぞれ紐づけて配置する。
- 入力されている奥行き・高さ・幅が収納ケースの最小サイズより小さかった場合、開始ボタンは無効化される
- それぞれの寸法入力欄の数値は、現在有効になっている数値ボタンの数をプラスボタン、マイナスボタンを押されたタイミングでそれぞれの寸法数値を加算・減算する。
‐ 寸法入力欄の数値は0以下にはならないように、計算結果が1未満の時は1に補正する。
- リセットボタンを押された時は対応する入力欄の数値を初期化する。
